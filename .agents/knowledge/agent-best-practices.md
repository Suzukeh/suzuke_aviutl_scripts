# AIコーディングエージェント運用ベストプラクティス

OpenCode で AviUtl2 スクリプト開発を効率的に進めるためのガイド。

## 1. 仕様駆動開発（最重要）

会話的な依頼ではなく、以下の4要素を含む **構造化された仕様** を渡す：

| 要素 | 内容 | 例 |
|------|------|-----|
| Goal | 何を作るか | 「@重力.tra に反発係数パラメータを追加する」 |
| Scope | 触るファイル／触らないファイル | 「@重力.tra のみ変更。他スクリプトには触れない」 |
| Context | 参考にすべきファイルや制約 | 「AviUtl2_Script_Guide.md のAPI仕様に従う」 |
| Acceptance | 完了の判定方法 | 「AviUtl2でスクリプトを読み込んで動作確認できること」 |

## 2. Planモード → Buildモード

- **複数ファイルにまたがる変更は必ずまずPlanモード（Tabキーで切替）で計画を立てる**
- 計画に納得したら Buildモードに切り替えて実装させる
- 単一ファイルの小さな変更は直接 BuildモードでもOK

## 3. 停止条件の明示

エージェントは「もっと良くしよう」と無限に手を入れ続ける傾向がある。以下を明示する：
- 「ファイルの編集が完了し、構文チェックが通ったら停止」
- 「それ以上の改善や追加機能の提案は不要」

## 4. 制約の優先順位付け（Must / Must Not / Prefer）

```
【必須】
- Lua 5.1 互換の記法を使う（LuaJIT）
- ファイルエンコーディングは UTF-8
- obj.getvalue / obj.setvalue は定義された変数インデックスのみ使う

【禁止】
- 他のスクリプトファイルを変更しない
- 新しい依存ライブラリを追加しない
- グローバル変数を新設しない（local限定）

【推奨】
- 計算量が多い処理はHLSLシェーダ化を検討
- ユーザー向けパラメータ名は日本語で
```

## 5. 検証可能な完了基準

「動けばOK」ではなく、検証可能な基準を指定する：
- 「AviUtl2でスクリプトをロードしてエラーが出ないこと」
- 「全パラメータがUIに表示されること」
- 「`luac -p` で構文エラーがないこと」
- 「既存の @重力.tra の挙動が変わっていないこと」

## 6. コンテキスト管理

| 置き場所 | 内容 |
|----------|------|
| `.agents/knowledge/` | プロジェクトの不変知識（スクリプト一覧、API仕様） |
| `.agents/progress.md` | 現在の進捗・引き継ぎ情報 |
| プロンプト | タスク固有の指示（Goal, Scope, Acceptance） |

**安定情報は毎回プロンプトに書かない。.agents/ 配下に置いて参照させる。**

## 7. エージェント固有の制御

| 機能 | 使い方 |
|------|--------|
| Plan agent | Tabキーで切替。分析・計画のみ。ファイル編集禁止 |
| Build agent | Tabキーで切替。実装モード。全ツール使用可 |
| @general | 複雑な調査・複数作業の並列実行 |
| @explore | コードベース探索（読み取り専用） |
| rules/style.md | 常時適用されるルール（日本語応答、可読性優先） |

## 8. エージェントの失敗パターンと対策

| 失敗パターン | 対策 |
|--------------|------|
| 存在しないAPIを捏造する | `AviUtl2_Script_Guide.md` にないAPIは使わないよう明示 |
| 不要なファイルに触る | Scopeで明示的に「触らないファイル」を指定 |
| 無限ループに陥る | 「n回試行してダメなら報告して停止」と指示 |
| 過剰な最適化・リファクタリング | 「依頼された変更以外は行わない」と明示 |
| 同じミスを繰り返す | `rules/` に失敗パターンを追加 |

## 9. 小さな変更単位

- 1タスク = 1ファイル変更が理想
- 複数ファイルにわたる場合は Planモードで分割計画を立てる
- `/undo` で簡単に戻せる単位で進める

## 10. レビューと品質

- エージェントの生成コードを鵜呑みにしない。必ず差分を確認する
- `luac -p` で構文チェック
- 既存機能のレグレッションがないか手動確認
- 問題があればパターン化して rules/ に追加
