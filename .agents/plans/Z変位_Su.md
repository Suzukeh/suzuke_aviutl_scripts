# Z変位_Su 実装計画

> 策定: 2026-07-17 / 最終更新: 2026-07-18

## 概要
高さマップ（輝度）に応じて画像を Z 方向（奥行き）に変位させるアニメーション効果 `.anm2`。
画像を N×M のメッシュに分割し、各頂点の Z を `高さ × 強さ` で変位させて `obj.drawpoly` で描画する。
AviUtl2 の 3D 空間上の実ジオメトリとなるため、オブジェクトの回転やカメラ制御に正しく追従する立体レリーフになる。

## 技術方針（検討結果）
| 案 | 判定 | 理由 |
|---|---|---|
| A. メッシュ分割 + `obj.drawpoly` (Lua) | **採用** | 真の3D変位が可能な唯一の手段 |
| B. ピクセルシェーダによるパララックス | 不採用 | 幾何は変わらず擬似3D。回転・カメラに正しく応答しない |
| C. 法線ライティングのみ（バンプ） | 不採用 | Z変位ではない（陰影オプションとして Phase 2 で検討） |
| 純 `.auf2` フィルタプラグイン (Rust) | 不可 | 画像フィルタAPI (get/set_image_data, shader等) のみで3D頂点をシーンに注入する手段が無い |
| `.mod2` スクリプトモジュール (Rust) | 補助 | 描画はLuaからしか行えないが、重い計算（サンプリング等）を高速化できる → Phase 2.5 |

## 面モード仕様
| モード | サンプリング | 見た目 |
|---|---|---|
| 連続面（既定） | 格子頂点 (分割数+1)² で取得、隣接quadと頂点Zを共有 | 面が滑らかに折れ曲がる |
| ブロック | セル 分割数² で取得、4頂点に同一Z | ピクセル積層 |
| ブロック(側面あり) | 同上 + 段差に壁ポリゴン | ソリッドな積層 |

Z変換式: `z = (基準 - 輝度) × 強さ`（正の強さで白が手前、負で反転）

## フェーズ構成
- **Phase 1 (完了)**: 純Lua MVP — 面モード3種、基本パラメータ（強さ/分割数/基準/ぼかし/カリング/透明スキップ）
- **Phase 1.5 (完了予定: 2026-07)**: 高速化 — ジオメトリキャッシュ + 純Luaループ改善
  - 詳細は `optimization.md` の Z-1 / Z-2 を参照
- **Phase 2**: マップ参照（レイヤー `obj.load("layer",n)` / 画像ファイル）、陰影シェーディング（高さ勾配→法線→簡易ライティング）、`obj.effect()` による後続フィルタ対応
- **Phase 2.5 (オプション)**: `.mod2` (Rust/aviutl2-rs) による高速化
  - Lua側: `getpixeldata` のポインタを `mod.heights(...)` に渡す
  - Rust側: RGBA解析→輝度→Z配列 `Vec<f64>` を返却（可能なら polys/walls 配列も構築して Lua のループを削減）
  - `pcall(require)` でモジュール不在時は純Lua処理にフォールバック
  - 詳細は `optimization.md` の Z-3 を参照
- **Phase 3**: ドキュメント整備、au2pkg 化（`.anm2`+`.mod2` は Script/ 配下に同梱可能）

## リスク・未検証事項
| リスク | 対策 |
|---|---|
| Lua `getpixel` ループのコスト | 縮小マップで点数を分割数に比例させる。重ければ `.mod2` 化 |
| 非平面quad（4隅のZが異なる）の描画品質 | 内部で三角形分割されるはず。対角折れ目はマップぼかし・分割数で緩和 |
| `drawpoly` のポリゴン数上限（不明） | 分割数上限192で実機確認 |
| 後続フィルタ効果が実行されない可能性 | 必要なら `obj.effect()` 対応（Phase 2） |
| 完全透明時の自動描画抑制 (`draw_state`) | 実機検証で確認 |

## 参照した既存実装
- `@立体化(g).anm2` — drawpoly の `4頂点×(x,y,z)+4×(u,v)` 形式、ポリゴン配列一括描画、culling、drawpoly呼出しによる自動描画抑制
- `引き伸ばし_Su.anm2` — tempbuffer への drawpoly 縮小描画
- `@DataEffect_Su.anm2` — `pixeloption("get","object")`+`getpixel`、cacheバッファ運用、`obj.effect("ぼかし")`
